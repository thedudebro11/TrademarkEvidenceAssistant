# User Journey

## Purpose

This document describes how a human experiences Trademark Evidence Assistant from first launch through completion.

It intentionally avoids implementation details.

If implementation decisions conflict with this document, the goal is to preserve the user experience whenever possible.

---

# Guiding Principle

The application should never feel like file management software.

It should feel like sitting down with an organized assistant that helps review evidence together.

The user should always understand:

• what they are looking at
• why it matters
• what information is missing
• what decision they are making
• what happens next

The application should reduce uncertainty.

Never create more.

---

# First Launch

The application opens.

No dashboard.

No statistics.

No charts.

No overwhelming navigation.

The interface is intentionally quiet.

The user immediately sees:

---------------------------------------

Trademark Evidence Assistant

Review your evidence one file at a time.

Nothing will modify your original files.

Select Evidence Folder

---------------------------------------

The first interaction should build trust.

---

# Selecting Evidence

The user selects:

workspaces/Fatletic/evidence/

The application validates:

✓ Folder exists

✓ Read permissions

✓ Original files will remain untouched

The user clicks:

Begin Scan

---

# Scanning

The scan should feel calm.

Never overwhelming.

The application communicates exactly what it is doing.

Examples:

Discovering files...

Reading metadata...

Calculating hashes...

Checking duplicates...

Creating Evidence Items...

The user always knows progress.

The scan never implies legal conclusions.

---

# Scan Complete

The application displays:

---------------------------------------

193 files discovered

193 Evidence Items created

Original files remain unchanged.

Review Ready

Start Reviewing

---------------------------------------

No unnecessary statistics.

The only important next action is:

Start Reviewing

---

# Review Queue

This is the heart of the application.

Everything else exists to support it.

The layout is intentionally simple.

Left:

Large preview.

Right:

Known metadata.

Suggested role.

Guided questions.

Evidence connections.

Current usefulness assessment.

Bottom:

Include

Maybe

Needs Follow-Up

Archive

Save & Next

The interface should encourage thoughtful review.

Never rushed review.

---

# Reviewing Evidence

Every file follows the same rhythm.

Observe.

Understand.

Answer.

Connect.

Decide.

Continue.

The application never asks irrelevant questions.

Every question includes a reason.

Example:

"When was this photo taken?"

Reason:

"This helps establish the timeline of brand use."

---

# Connections

Whenever appropriate the application suggests related evidence.

Example:

Possible related items:

• Printful Order

• Customer Photo

• Instagram Post

• Shipping Record

The software never creates legal conclusions.

The user confirms relationships.

---

# Progress

Progress should feel encouraging.

Avoid:

193 files remaining

Prefer:

---------------------------------------

27 reviewed

18 included

6 follow-up

Estimated time remaining:

42 minutes

---------------------------------------

Completion should always feel achievable.

---

# Completing Review

When every item has been reviewed:

---------------------------------------

Review Complete

193 Evidence Items reviewed.

Ready to prepare your trademark evidence package.

---------------------------------------

The application now offers:

Generate Evidence Package

Generate Evidence Binder

Review Follow-Up Items

Nothing more.

---

# Export

The export process clearly explains:

Original evidence remains untouched.

Selected evidence will be copied into a new organized package.

Hashes will be preserved.

The application never hides filesystem actions.

---

# Completion

The application finishes with:

Evidence Package Generated

Evidence Binder Generated

Export Complete

The user should feel confident that:

Nothing was modified.

Everything is documented.

Everything is organized.

Everything is traceable.

The software's job is complete.
